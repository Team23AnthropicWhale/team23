# SafeCase

Mobile-first case management for child protection field workers. Built for low-connectivity environments where cases must be captured, encrypted, and synced without a reliable internet connection.

## What it does

Field workers record child protection cases on their phones using structured forms (consent, registration, assessment, case plan, review, closure). Cases are stored encrypted on-device and synced to a central server when connectivity is available, or directly to a supervisor's device over Bluetooth when it is not.

Supervisors can review cases, assign tasks, and push org-wide key updates to caseworker devices — all without requiring a live server connection.

## Structure

```
backend/         Laravel 13 REST API — case storage, auth, user management
frontend/        Expo (React Native) mobile app — forms, dashboard, sync UI
secure-sync/     Encryption and connectivity layer
  lookup.ts            Compact field-value codes — shrinks records before encryption
  cryptoCore.ts        Platform-agnostic factory for compress → encrypt pipeline
  cryptoProvider.ts    pako + Web Crypto AES-256-GCM implementations
  keyManager.ts        PBKDF2 key derivation, secure storage, ECDH session keys, org-key transfer
  bleProtocol.ts       BLE wire types, chunking, reassembly, per-message encryption
  bleConnection.ts     BleManager wrapper — scan, connect, MTU, write, monitor, retransmit
  syncOrchestrator.ts  Pairing flow, role negotiation, bidirectional case replication
forms/           Official CPMS paper forms (source of truth for field definitions)
```

## Security model

- All case data is encrypted with AES-256-GCM before leaving memory
- Keys are derived from a user PIN via PBKDF2 (100k iterations, SHA-256) and stored in device Keychain / Keystore
- A single org key is generated by the supervisor and transferred to caseworker devices over Bluetooth, wrapped with a shared password and HMAC-authenticated
- Bluetooth transfers use 4-digit verbal PIN pairing and per-message AES-GCM encryption
- Conflicts are flagged for human review — never auto-resolved

## Getting started

**Backend**
```sh
cd backend
cp .env.example .env
composer install
php artisan key:generate
php artisan migrate --seed
php artisan serve
```

**Frontend**
```sh
cd frontend
npm install
npx expo start
```
